Proiect ierbar digital

Introducere

  1. Scopul produsului

    Scopul produsului este de a oferi utilizatorilor o aplicație web de tip ierbar digital, care să permită organizarea și gestionarea colecțiilor de plante presate și atributelor aferente. Aceasta va oferi o serie de facilități de căutare multi-criterială și va permite utilizatorilor să genereze, recomande și partajeze albume tematice, precum plante medicinale sau flori din zona montană si multe altele.

  2. Audienta tinta

    Audienta vizata de aceasta aplicatie web de tip ierbar digital poate fi formata din persoane interesate de botanica, plante si flori, sau de utilizatori care doresc sa organizeze si sa-si gestioneze colectiile de imagini. De asemenea, aplicatia poate fi utila si pentru fotografi, mai ales pentru cei pasionati de natura si de fotografia de plante.

    • Unsplash API
    • W3Schools HTML
    • MDN Web Docs: Learn HTML
    • Imgur API

Descriere generală

  1. Descrierea produsului

    Această aplicație web oferă posibilitatea utilizatorilor de a organiza și gestiona colecțiile de plante. Utilizatorii pot căuta plante în funcție de diverse criterii, genera albume tematice. Aplicația oferă o interfață responsive si intuitiv de utilizat.

  2. Funcționalități produsului

    • Căutare multi-criterială a plantelor
    • Generarea de albume tematice
    • Generarea de rapoarte statistice despre plante in formate PDF si CSV
    • Clasamentul celor mai populare plante colectate

  3. Categorii de utilizatori

    • Colecționari de plante
    • Studenți sau cercetători în domeniul botanicii
    • Pasionați de plante și grădinărit

  4. Mediu de operare

    Aplicația este disponibilă pe platformele desktop, putând fi accesată prin intermediul unui browser web.

  5. Design și constrângeri de implementare

    Aplicația va fi dezvoltată folosind HTML, CSS și JavaScript, iar datele vor fi stocate într-o bază de date MongoDB. Interfața va fi creată cu ajutorul vanilla JS și va fi responsivă pentru a se adapta la diferite dimensiuni de ecran.

    Culori predominante utilizate: white green black

  6. Dependențe

    • API-ul public Imgur pentru a uploadarea pozelor de catre utilizatori
    • API-ul public Unsplash pentru a obține fotografii ale plantelor
    • Baza de date MongoDB pentru a stoca datele
    • JavaScript
    • NodeJS
    • HTML
    • CSS

Cerințele Interfeței

Pentru a asigura o utilizare facilă a produsului, interfața trebuie să îndeplinească următoarele cerințe:

  • Design simplu și intuitiv
  • Interfață responsivă și adaptabilă la diferite dispozitive
  • Posibilitatea de a filtra și căuta plante
  • Posibilitatea de a crea albume tematice
  • Posibilitatea de a uploada o imagine
  • Posibilitatea de a vizualiza statistici despre colecție
  • Posibilitatea de a aprecia o fotografie
  • Posibilitatea de a vizualiza albumele, aprecierile si upload-urile

Interfețe Utilizator

Interfața utilizator este formată din următoarele pagini:

  • Pagina principală - unde se pot căuta plante, și se pot accesa restul paginilor.
  • Pagina profil - unde se pot crea albume propii, vizualiza imaginile apreciate, informatii despre utilizator, vizualizare albumelor, upload de imagini, editare a profiluli
  • Pagina catolog - se pot vizualiza plantele de un anumit tip sau criteriu prin intermediul acestei pagini putem adauga fotografii in colectiile noastre si in sectiunea de like
  • Pagina de vizualizare a statisticilor - unde se pot vizualiza diverse statistici despre categorii de plante, cum ar fi cele mai populare plante.

Caracteristicile Sistemului

  1. Home Page

    Aceasta reprezinta pagina de start a aplicatiei ce contine:

    • Butoane pentru autentificare si inregistrare
    • Buton de profil
    • Linkuri catre anumite colectii predefinite
    • Bara de cautare pentru plantele descrise
    • Sectiune despre proiect
    • Sectiune pentru contact

  2. Login and Register Page

    Aceasta sectiune va contine formularul de autentificare a utilizatorilor si orice alte informatii relevante, inclusiv:

    • Camp pentru email si parola
    • Link de uitare a parolei
    • Buton de submit
    • Buton de redirectionare catre campul de inregistrare
    • Link de intoarcere la pagina de start

    Aceasta sectiune va contine formularul de inregistrare a utilizatorilor si orice alte informatii relevante, inclusiv:

    • Camp pentru nume, e-mail si parola
    • Buton de submit
    • Link de intoarcere la pagina de start
    • Buton de redirectionare catre campul de logare
    În implementarea paginilor de autentificare și înregistrare, au fost utilizate următoarele tehnologii și biblioteci:
    • JWT (JSON Web Tokens):
      1. Pentru a asigura securitatea și integritatea datelor utilizatorilor, am folosit tehnologia JSON Web Tokens (JWT) pentru generarea unui token de autentificare.
      2. Parola introdusă de utilizator a fost criptată înainte de a fi stocată în baza de date sau trimisă către server pentru verificare. Astfel, parolele utilizatorilor sunt păstrate în siguranță în cazul unui posibil compromis al sistemului.
    • Bcrypt:
      1. Pentru criptarea parolelor utilizatorilor, am folosit biblioteca Bcrypt.
      2. Bcrypt este o bibliotecă de hashing pentru parolă care adaugă un nivel suplimentar de securitate prin aplicarea unui algoritm puternic de hashing și prin adăugarea unui "salt" aleatoriu la parolă înainte de a o cripta.
      3. Utilizarea lui Bcrypt ajută la prevenirea atacurilor de tip brute force și la protejarea parolelor utilizatorilor în cazul în care baza de date este compromisă.

  3. Rezultate căutare (pagina Catalog)

    • Numele plantei: Numele plantei
    • Taxonomia: Familia / Genul / Specia
    • Descriere: Descriere scurta a plantei
    • Imagine: img
    • Alte detalii preluate din Unsplash
    In pagina de cautare vom avea niste albume predefinite practic aceste albume sunt mai multe fotografii care au un tag comun. Pe baza acestor taguri putem realiza cautare criteriala. Utilizatorul poate alege mai multe taguri dupa care sa faca filtrarea. In urma acestei actiuni utilizatorul va primi poze cu plantele respective. Toate plantele sunt stocate in baza de date si trimise pe front pe baza unor servicii. User-ul poate adauga o poza in sectiunea de liker-uri si colectii. Acesta isi poate crea colectii personalizate si poate adauga plante.

  4. Statistici plante

    • Numarul de plante a unui tag
    • Cele mai populare taguri de plante dupa likes
    • Cele mai populare taguri de plante dupa views
    In aceasta pagina utilizatorul poate descarca statistici in formate PDF si CSV. Pentru preluarea datelor am folosit modelul din baza de date unde stocam toate plantele

  5. Pagina de profil

    • Poza de profil si de coperta
    • O mica descriere personaliza
    • Numarul de like-uri date, numarul de colectii create si numarul de imagini uploadate
    • Sectiune de creare colectie
    • Sectiune de edit profile (profile photo, cover photo, description, email)
    Aceasta pagina este destinata user-ului. El si-o poate customiza in functie de cum doreste. In baza de date fiecare user este stocat alaturi de cateva informatii precum email, descriere, un array de colectii, array de imagini apreciate, imagini postate si multe alte informatii necesare pentru gestionarea serviciilor.
